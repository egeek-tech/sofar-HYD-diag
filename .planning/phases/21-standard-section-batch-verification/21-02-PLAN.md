  <action>
Parse `tools/section-sweep/results.json` and process each section.

**For each section, apply the Phase 20 removal pattern (per D-04, D-05):**

1. **Identify failing probes:** Any probe in results.json with `"status": "FAIL"` or `"status": "ERROR"` must be removed from the source definition file.

2. **Remove individual probes:** Delete the `Probe{...}` struct literal line from the source file. The probe is identified by its `Addr` field matching the hex address in results.json.

3. **Remove empty groups:** If a ProbeGroup has zero remaining probes after removal, delete the entire `ProbeGroup{...}` struct literal from the source file (per D-05).

4. **Remove empty sections from hub registration:** If ALL groups in a section's variable are removed (variable becomes empty slice), also remove the `h.RegisterGroupedSection("name", register.XxxGroups)` line from `internal/hub/hub.go` at line 640-650. This prevents registering an empty section. NOTE: Only do this if the entire variable is empty.

5. **Handle PV specially:** If PV probes fail, the fix is in `GeneratePVGroups()` function body (removing the failing probe from the per-channel loop or the Total PV Power group). If only channels > 2 fail but channels 1-2 pass, no removal needed (the default 2 channels are correct).

6. **Update test assertions in register_test.go:**
   - **TestGridGroups (line 252):** If any Grid groups or probes were removed:
     - Update `len(GridGroups) != 7` to match new count
     - Update `expectedNames` slice to remove deleted group names
     - Update per-group probe count assertions and address assertions
   - **TestEPSGroups (line 354):** Same pattern -- update group count, names, probe assertions
   - **TestGeneratePVGroups2 (line 645):** Update if any PV probes removed
   - **Meter/DCDC/PCU/BDU:** No existing tests to update. If probes are removed from these sections, no test changes needed (there are no tests for them).

7. **If results.json shows zero failures across all sections:** No source changes needed. Commit results.json only and mark phase complete -- all sections already work on hardware.

8. **Add sweep verification comment** at top of any modified source file (following Phase 20 pattern):
   ```go
   // Hardware-verified via tools/section-sweep (2026-04-15).
   // Registers returning illegal data address removed.
   ```
