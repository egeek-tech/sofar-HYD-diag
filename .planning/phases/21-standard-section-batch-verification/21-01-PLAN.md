**Structure (follow config-sweep/main.go exactly for shared patterns):**

1. **Build tag and package:** `//go:build section_sweep` on line 1, `package main`.

2. **Imports:** Same as config-sweep (`encoding/json`, `flag`, `fmt`, `log`, `os`, `strings`, `time`) plus `sofar-hyd-diag/internal/modbus` and `sofar-hyd-diag/internal/register`.

3. **Result types (JSON output schema per D-06):**

```go
type SweepResult struct {
    Timestamp  string          `json:"timestamp"`
    Host       string          `json:"host"`
    Port       int             `json:"port"`
    SlaveID    int             `json:"slave_id"`
    PVChannels int             `json:"pv_channels"`
    Sections   []SectionResult `json:"sections"`
    Summary    SummaryResult   `json:"summary"`
}

type SummaryResult struct {
    TotalSections int `json:"total_sections"`
    TotalProbes   int `json:"total_probes"`
    Passed        int `json:"passed"`
    Failed        int `json:"failed"`
    Errors        int `json:"errors"`
}

type SectionResult struct {
    Name        string       `json:"name"`
    TotalProbes int          `json:"total_probes"`
    Passed      int          `json:"passed"`
    Failed      int          `json:"failed"`
    Errors      int          `json:"errors"`
    Spans       []SpanResult `json:"spans"`
}

type SpanResult struct {
    StartAddr   string        `json:"start_addr"`
    TotalCount  uint16        `json:"total_count"`
    BatchStatus string        `json:"batch_status"`
    Probes      []ProbeResult `json:"probes"`
}

type ProbeResult struct {
    Name   string `json:"name"`
    Group  string `json:"group"`
    Addr   string `json:"addr"`
    Count  uint16 `json:"count"`
    Status string `json:"status"`
    Error  string `json:"error,omitempty"`
}
```

4. **CLI flags (same as config-sweep):**
   - `-host` (required, string, default "")
   - `-port` (int, default 4192)
   - `-slave` (int, default 1)
   - `-pv` (int, default 2) -- PV channel count, per Claude's discretion
   - Usage message: `"Usage: section-sweep -host <inverter-ip> [-port 4192] [-slave 1] [-pv 2]\n"`

5. **Connection setup:** Identical to config-sweep lines 57-64:
   ```go
   addr := fmt.Sprintf("%s:%d", *host, *port)
   conn, err := modbus.Connect(addr)
   // ... defer conn.Close(), logger := modbus.DiscardLogger()
   ```

6. **Section definitions (per D-02 -- single invocation, all 7 sections):**
   ```go
   type sectionDef struct {
       name   string
       groups []register.ProbeGroup
   }
   sections := []sectionDef{
       {"grid", register.GridGroups},
       {"eps", register.EPSGroups},
       {"pv", register.GeneratePVGroups(*pvChannels)},
       {"meter", register.MeterGroups},
       {"dcdc", register.DCDCGroups},
       {"pcu", register.PCUGroups},
       {"bdu", register.BDUGroups},
   }
   ```

7. **Main sweep loop (per D-03 -- batch spans only, per D-07 -- fallback on batch failure):**
   For each section:
   a. Call `register.AnalyzeBatchPlan(sec.groups)` to compute spans.
   b. For each span in `plan.Spans`:
      - Attempt batch read: `modbus.ReadHoldingRegistersTCP(conn, logger, byte(*slaveID), span.StartAddr, span.TotalCount)`
      - If batch succeeds: mark `batch_status = "PASS"`, all probes in span get status `"PASS"`.
      - If batch fails: mark `batch_status = "FAIL"`, then iterate `span.Probes` and read each individually:
        - `modbus.ReadHoldingRegistersTCP(conn, logger, byte(*slaveID), pm.Probe.Addr, pm.Probe.Count)`
        - Apply 3-way error classification (identical to config-sweep lines 94-113):
          - `err == nil` -> `"PASS"`
          - `strings.Contains(err.Error(), "err=0x02")` -> `"FAIL"` with error `"illegal data address (0x02)"`
          - default -> `"ERROR"` with error string, close conn and reconnect via `modbus.Connect(addr)`
        - Apply 500ms delay after each individual read.
      - Apply 500ms delay after each batch read (both success and failure-with-fallback).
   c. Print progress to stderr: `fmt.Fprintf(os.Stderr, "[%s] %s / %s (0x%04X)\n", status, sec.name, probeName, probeAddr)`

8. **Important: connection variable must be reassignable.** The `conn` variable is reassigned on reconnect. Do NOT use `defer conn.Close()` at the top level -- instead close in a deferred function that captures the final value, or close explicitly at the end. Follow the config-sweep pattern where `conn` is a `net.Conn` variable that gets reassigned on reconnect.

9. **JSON output (to stdout):** Build `SweepResult` with all section results and summary totals. Use `json.MarshalIndent(result, "", "  ")` and `fmt.Println(string(out))`.

10. **Summary to stderr:** `fmt.Fprintf(os.Stderr, "\nSweep complete: %d sections, %d passed, %d failed, %d errors\n", ...)`.
