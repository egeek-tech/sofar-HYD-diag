**CLI flags** (matching project convention from main.go and xlsx-discover):

**JSON output types:**
```go
type SweepResult struct {
    Timestamp      string        `json:"timestamp"`
    Host           string        `json:"host"`
    Port           int           `json:"port"`
    SlaveID        int           `json:"slave_id"`
    TotalRegisters int           `json:"total_registers"`
    Passed         int           `json:"passed"`
    Failed         int           `json:"failed"`
    Errors         int           `json:"errors"`
    Groups         []GroupResult `json:"groups"`
}

type GroupResult struct {
    Name   string        `json:"name"`
    Probes []ProbeResult `json:"probes"`
}

type ProbeResult struct {
    Name   string `json:"name"`
    Addr   string `json:"addr"`      // hex format "0x1234"
    Count  uint16 `json:"count"`
    Status string `json:"status"`    // "PASS", "FAIL", or "ERROR"
    Error  string `json:"error,omitempty"`
}
```

**Main logic:**
1. Parse flags, validate host is non-empty.
2. Connect via `modbus.Connect(fmt.Sprintf("%s:%d", *host, *port))`. Fatal on error (matching xlsx-discover pattern).
3. Create a logger: `modbus.DiscardLogger()` (suppress verbose modbus logging; progress goes to stderr).
4. Count total probes by iterating `register.ConfigurationGroups` before the sweep begins. Print "Sweeping N registers across M groups..." to stderr.
5. For each group in `register.ConfigurationGroups`:
   - For each probe in group.Probes:
     - Call `modbus.ReadHoldingRegistersTCP(conn, logger, byte(*slaveID), probe.Addr, probe.Count)`
     - Classify result:
       - `err == nil` -> status "PASS"
       - `strings.Contains(err.Error(), "err=0x02")` -> status "FAIL", error "illegal data address (0x02)"
       - Any other error -> status "ERROR", error `err.Error()`. On ERROR, attempt reconnection: `conn.Close()` then `conn, err = modbus.Connect(addr)`. If reconnect fails, `log.Fatalf`. If reconnect succeeds, continue from next probe.
     - Print progress to stderr: `fmt.Fprintf(os.Stderr, "[%s] %s / %s (0x%04X)\n", status, group.Name, probe.Name, probe.Addr)`
     - Sleep 500ms between reads: `time.Sleep(500 * time.Millisecond)` (project convention; the broker is not used so the tool must enforce this itself per Pitfall 4 in RESEARCH.md)
6. After sweep completes, build `SweepResult` struct with timestamp (`time.Now().UTC().Format(time.RFC3339)`), connection params, tallies, and per-group results.
7. Marshal to JSON with `json.MarshalIndent(result, "", "  ")`.
8. Write JSON to stdout (so user can redirect to file: `go run -tags config_sweep ./tools/config-sweep/ -host 192.168.1.100 > results.json`).
9. Print summary to stderr: `fmt.Fprintf(os.Stderr, "\nSweep complete: %d passed, %d failed, %d errors\n", passed, failed, errors)`.

**Connection management for long sweeps** (Pitfall 5 in RESEARCH.md):
The sweep reads ~210 registers at 500ms each (~105 seconds). If a read returns a non-0x02 error (timeout, connection reset), reconnect and continue. Do NOT retry the failed register on reconnect -- just record it as ERROR and move to the next. This avoids infinite retry loops on flaky connections.

**Important implementation details:**
- Addr in JSON output formatted as hex string `fmt.Sprintf("0x%04X", probe.Addr)` for human readability and grep-ability.
- The tool reads each register INDIVIDUALLY (not in batches) per Pitfall 1 in RESEARCH.md. Each probe is one `ReadHoldingRegistersTCP` call.
- No need to set read/write deadlines manually -- `ReadHoldingRegistersTCP` handles its own deadlines internally (10s read, 3s write per tcp.go).
