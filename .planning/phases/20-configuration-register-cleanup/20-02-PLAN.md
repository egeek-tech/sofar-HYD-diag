  <action>
Parse `tools/config-sweep/results.json` and use it to surgically edit `internal/register/configuration.go` (per D-03 static source removal).

**Step 1: Identify removals from JSON.**
Read results.json. For each probe with `"status": "FAIL"`, note the group name and hex address. Build two lists:
- `probes_to_remove`: list of (group_name, addr_hex) pairs
- `groups_to_remove`: groups where ALL probes have status "FAIL" (per D-04)

**Step 2: Remove individual failing probes.**
For each probe in `probes_to_remove` that is NOT in a fully-removed group (those are handled in step 3):
- Find the Probe struct literal in configuration.go with matching `Addr: 0xNNNN`
- Delete the entire line (or multi-line block if the probe spans lines)
- Do NOT change any surrounding probes -- leave them exactly as they are

**Step 3: Remove entirely-empty groups.**
For each group in `groups_to_remove`:
- If the group is an inline definition in `buildConfigurationGroups()`: delete the entire `{Name: "Group Name", Probes: []Probe{...}},` block
- If the group is one of the EMS Time Period groups from `emsTimePeriodGroups()`: remove the corresponding `periodDef` entry. If ALL 6 periods are removed, delete the entire `emsTimePeriodGroups()` function and the `groups = append(groups, emsTimePeriodGroups()...)` line
- If the group is a Safety group from `safetyGroups()`: remove the corresponding group entry from the returned slice. If ALL safety groups are removed, delete the entire `safetyGroups()` function and the `groups = append(groups, safetyGroups()...)` line
- If all Network Config probes fail: delete the `networkConfigGroup()` function and the `groups = append(groups, networkConfigGroup())` line

**Step 4: Add removal documentation comment.**
At the top of `buildConfigurationGroups()`, after the existing doc comment for `ConfigurationGroups`, add:
```go
// Registers verified against real hardware on YYYY-MM-DD using tools/config-sweep.
// Probes returning Modbus exception 0x02 (illegal data address) have been removed.
// See tools/config-sweep/results.json for the full sweep report.
```
Use the actual date from results.json timestamp.

**Step 5: Verify no orphaned enum maps** (Pitfall 6 in RESEARCH.md).
After editing, check if any enum maps in configuration.go are now unreferenced. Go does NOT error on unused package-level variables, so this is a code cleanliness check, not a build requirement. If an enum is only referenced by removed probes, add a comment `// Note: {EnumName} may be unused after cleanup` but do NOT delete it (it may be used elsewhere).

**Key rules:**
- Only remove probes with status "FAIL" in the sweep results. Never remove "PASS" or "ERROR" probes.
- "ERROR" probes (transient failures like timeouts) must be KEPT -- they may work on a stable connection.
- Do not edit batch.go, hub.go, or any file other than configuration.go -- batch plans auto-recompute (per RESEARCH.md anti-patterns).
- Do not add any runtime filtering mechanism -- this is static removal per D-03.
