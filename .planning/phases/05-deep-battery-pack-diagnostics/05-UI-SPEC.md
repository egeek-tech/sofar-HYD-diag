# Phase 5 -- UI Design Contract

> Visual and interaction contract for deep battery pack diagnostics: hierarchical pack navigation via bitmap click or dropdown selectors, pack detail sub-view with breadcrumb navigation, 24-cell voltage grid with deviation color-coding, pack temperatures with thermal range indicators, and alarm/protection/fault/balance bitmap decoding. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

### Pack Selector Dropdowns (D-07)

Three `<select>` elements in the content header bar. These replace/extend the Phase 4 topology dropdowns when a pack is being viewed. They share the same position and styling as the Phase 4 topology controls.

| Dropdown | Label | Options | Constrained by |
|----------|-------|---------|----------------|
| Input | "Input:" | 1 to configured inputs | BAT_INPUTS topology value |
| Tower | "Tower:" | 1 to configured towers | BAT_TOWERS topology value |
| Pack | "Pack:" | 1 to configured packs | BAT_PACKS topology value |

**Styling:** Reuse `.topology-controls`, `.topology-label`, `.topology-select` CSS classes from Phase 4. Same dimensions and behavior.

**Behavior:**
1. All three dropdowns visible when pack detail sub-view is active
2. Selecting a new pack triggers the 0x9020 write-read cycle (D-02)
3. While loading, show loading indicator in pack detail content area
4. Dropdown values constrained to topology config. If topology changes, dropdowns update ranges
5. Values NOT persisted to localStorage (pack selection is transient, per D-02 on-demand only)

#### Cell Color Coding (D-09)

Deviation is computed as `abs(cell_voltage - average_voltage)`.

| Deviation | Background | Text Color | CSS Class |
|-----------|-----------|------------|-----------|
| <= 5mV | `#e8f5e9` (`--cell-good`) | `#2e7d32` (`--cell-good-text`) | `.cell--good` |
| 5-20mV | `#fff8e1` (`--cell-warn`) | `#e65100` (`--cell-warn-text`) | `.cell--warn` |
| > 20mV | `#ffebee` (`--cell-danger`) | `#c62828` (`--cell-danger-text`) | `.cell--danger` |

Color coding applies to the voltage value text and the cell background. Cell number label remains `#888888` regardless.

### Cell Voltage Computation (frontend)

```
1. Receive 24 cell voltage values (0x9051-0x9068, each in 0.001V)
2. Compute average = sum / 24
3. For each cell, deviation_mV = abs(cell_mV - average_mV)
4. Find min/max cell index and value from the 24 cells
5. Compute spread = max - min (in mV)
6. Verify: compare computed min against 0x906A, computed max against 0x9069
7. Apply color class per deviation thresholds
```

## Interaction Contract

### BMS Section Navigation (extends Phase 4)

The BMS section now has two view modes:

1. **BMS Overview** (Phase 4 behavior, enhanced with clickable bitmap)
2. **Pack Detail** (new in Phase 5)

State transitions:

| From | Action | To |
|------|--------|----|
| BMS Overview | Click online bitmap cell | Pack Detail (loading) |
| BMS Overview | Click offline bitmap cell | BMS Overview (inline warning) |
| Pack Detail | Click "Back to BMS" button | BMS Overview |
| Pack Detail | Click breadcrumb "Input N" or "Tower M" | BMS Overview |
| Pack Detail | Click breadcrumb "Battery" | Battery section (navigate) |
| Pack Detail | Change pack selector dropdown | Pack Detail (new pack, loading) |
| Pack Detail | Navigate to different section via sidebar | Other section (clears pack state) |
| Any section | Click BMS in sidebar | BMS Overview (not pack detail) |

### Pack Detail Rendering Algorithm

On receiving `pack_data` message:

1. Clear pack detail content area
2. Render breadcrumb bar with current input/tower/pack
3. For each group in `groups`:
   a. If `type === "cell_grid"` -- delegate to `renderCellVoltageGrid(group)` (new)
   b. If `type === "pack_status"` -- delegate to `renderPackStatus(group)` (new, extends fault-card pattern)
   c. If `type === "balance"` -- delegate to `renderBalanceState(group)` (new)
   d. Otherwise -- use existing `renderGroupCard(group)` for standard key-value display
4. For temperature group: apply color classes based on `temp_raw` values
5. Render timestamp
6. Trigger success flash
7. Update bitmap grid selected cell highlight (if bitmap is visible)

### Auto-Refresh in Pack Detail

When auto-refresh is active and user is viewing a pack detail:
- Server re-triggers the 0x9020 write-read cycle every 10s for the selected pack
- Pack detail view updates in-place (same as section data refresh pattern)
- Success/error flash applies to the pack detail content area

### Topology Dropdown Interaction in Pack Detail

If user changes topology via the topology dropdowns (e.g., reducing towers from 4 to 2):
- If currently viewed pack would be out of range, return to BMS overview with a warning
- Pack selector dropdown ranges update to match new topology

## Accessibility Baseline

Inherited from Phase 3/4 with additions:

| Concern | Implementation |
|---------|----------------|
| Cell voltage communication | Deviation severity communicated via both color AND position relative to summary values. Cell number visible in each cell. Color is supplementary to numeric comparison |
| Temperature communication | Temperature severity communicated via both color AND numeric value. User can assess danger from the number itself; color is supplementary |
| Alarm/fault communication | Status communicated via both color AND text descriptions. Checkmark/warning icons + text labels. Same pattern as Phase 3 fault card |
| Breadcrumb navigation | Clickable segments are `<button>` or `<a>` elements. Current location (last segment) is visually distinct (no underline, different color). Separator is decorative text |
| Bitmap interactivity | Clickable cells are `<button>` elements (not `<div>`) for keyboard accessibility. Online/offline state communicated via color + position in grid + legend. Selected state has distinct visual (blue, thicker border) |
| Pack selector dropdowns | Standard `<select>` elements with preceding text labels. Same pattern as Phase 4 topology dropdowns |
| Color contrast | Cell good: `#2e7d32` on `#e8f5e9` = 5.5:1 (AA). Cell warn: `#e65100` on `#fff8e1` = 5.1:1 (AA). Cell danger: `#c62828` on `#ffebee` = 7.8:1 (AAA). Spread danger `#c62828` on white = 8.0:1 (AAA). Bitmap selected: white on `#1565c0` = 5.5:1 (AA) |
| Screen reader | Cell grid cells include aria-label: "Cell N: X.XXXV, deviation NmV". Pack status card uses role="status" for live updates |
