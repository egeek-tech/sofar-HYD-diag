**Types:**
```go
// ProbeMapping maps a single probe to its byte position within a batch response.
type ProbeMapping struct {
    Probe      Probe
    GroupName  string
    ByteOffset int // (probe.Addr - span.StartAddr) * 2
    ByteLength int // probe.Count * 2
}

// BatchSpan represents a contiguous range of registers readable in one Modbus request.
type BatchSpan struct {
    StartAddr  uint16
    TotalCount uint16
    Probes     []ProbeMapping
}

// BatchPlan is the complete batch read strategy for a section.
type BatchPlan struct {
    Spans       []BatchSpan
    Unbatchable []ProbeMapping
}
```

**Function: AnalyzeBatchPlan(groups []ProbeGroup) BatchPlan**

Algorithm (per D-01, D-02, D-07, D-08):
1. Flatten all probes from all groups into annotatedProbe structs (probe + groupName).
2. Filter: probes with Count == 0 go to Unbatchable (synthetic probes like system time 0x042C).
3. Sort remaining by Addr ascending.
4. Walk sorted probes building spans:
   - For each probe, compute `nextAddr = currentSpan.StartAddr + currentSpan.TotalCount` (the "end" of the current span).
   - If `probe.Addr == nextAddr`: contiguous. Check if extending would exceed MaxBatchRegisters. If yes, close current span and start new. If no, extend: `currentSpan.TotalCount += probe.Count`, add ProbeMapping.
   - If `probe.Addr > nextAddr`: gap. Close current span, start new span at this probe.
   - If `probe.Addr < nextAddr`: OVERLAP. The probe starts inside the current span's range. Add the ProbeMapping with correct ByteOffset = `(probe.Addr - currentSpan.StartAddr) * 2`. If `probe.Addr + probe.Count > nextAddr`, extend TotalCount to `probe.Addr + probe.Count - currentSpan.StartAddr`. Otherwise, TotalCount unchanged.
5. Close final span.
6. Return `BatchPlan{Spans: spans, Unbatchable: unbatchable}`.

ByteOffset for each ProbeMapping: `int(probe.Addr - span.StartAddr) * 2`
ByteLength for each ProbeMapping: `int(probe.Count) * 2`

Import only `"sort"` from stdlib. No I/O, no logging, no side effects. This is a pure function in a pure data package.

Handle edge cases:
- Empty groups: return `BatchPlan{}` (zero spans, zero unbatchable)
- All probes synthetic: return `BatchPlan{Unbatchable: ...}` (zero spans)
- Single probe: return `BatchPlan{Spans: [{probe}]}` (one span)
- Exactly 60 contiguous regs: one span (not split)
- 61 contiguous regs: two spans (60 + 1)
