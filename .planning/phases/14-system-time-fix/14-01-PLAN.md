Purpose: Users see human-readable time instead of 6 raw register values, and the batch read saves 5 Modbus reads (2.5 seconds) per refresh cycle.

Replace the current tooltip line construction (lines 1047-1051):
```javascript
var lines = [];
// D-19: Omit Register line if addr is 0x0000 (composed values)
```

With:
```javascript
var lines = [];
if (addr && addr !== '0x0000') {
    if (raw && raw.indexOf(' | ') !== -1) {
        // D-08: Register range tooltip for composed values (e.g., System time)
        var parts = raw.split(' | ');
        lines.push('Registers: ' + parts[0]);
        lines.push('Raw: ' + parts[1]);
        raw = null; // consumed, prevent duplicate Raw line below
    } else {
        lines.push('Register: ' + addr);
    }
}
if (raw) lines.push('Raw: ' + raw);
if (time) lines.push('Last read: ' + time);
```

This logic:
- Checks if `raw` contains the ` | ` delimiter (pipe with spaces)
- If yes: splits into range string (e.g., "0x042C-0x0431") and raw values (e.g., "26, 4, 13, 10, 59, 34"), showing "Registers: 0x042C-0x0431" on line 1 and "Raw: 26, 4, 13, 10, 59, 34" on line 2
- Sets `raw = null` after consuming to prevent the fallthrough `if (raw)` from adding a duplicate "Raw:" line
- If no pipe: preserves existing behavior exactly (single "Register: 0x042C" line + "Raw: value" line)
- The "Last read:" timestamp line remains unchanged
