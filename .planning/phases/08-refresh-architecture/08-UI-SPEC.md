## Component Inventory (Phase 8 Scope)

This phase modifies 3 UI elements in the content header controls area. No new pages, no new sections, no new cards.

### 1. Cycle Delay Dropdown

**Source:** D-05, D-06, D-07, D-08

| Property | Value |
|----------|-------|
| Element | `<select id="cycle-delay-select">` |
| Class | `.cycle-delay-select` |
| Height | 32px (matches existing `.pv-channel-select`) |
| Horizontal padding | `var(--space-sm)` (8px) |
| Font size | `var(--font-size-body)` (14px) |
| Font family | `var(--font-family)` |
| Text color | `var(--color-text)` (#333333) |
| Background | `var(--color-card)` (#ffffff) |
| Border | `1px solid #cccccc` |
| Border radius | `var(--border-radius)` (4px) |
| Focus border | `var(--color-accent)` (#4a90d9) |
| Cursor | `pointer` |
| Transition | `border-color var(--transition-speed) ease` (200ms) |
| Visibility | Hidden (`display:none`) until connected to a section |
| Position | Inline within `.content__header-controls`, immediately before the auto-refresh button |

**Options:**

| Value (ms) | Label |
|------------|-------|
| `0` | `Continuous` (default, selected) |
| `5000` | `5s` |
| `10000` | `10s` |
| `30000` | `30s` |

**Persistence:** Value saved to `localStorage` key `sofar_cycle_delay`. On page load, restore saved value; if none saved, default to `0` (Continuous).

**CSS rule:** Reuse existing `.pv-channel-select` class exactly. The cycle delay dropdown uses the same visual treatment. No new CSS class needed unless the executor determines the existing class name is semantically misleading; in that case, create `.cycle-delay-select` with identical rules.

### 2. Auto-Refresh Button (Modified)

**Source:** D-09, D-10, D-13

| Property | Value |
|----------|-------|
| Element | `<button id="btn-auto-refresh">` |
| Class | `.btn-auto-refresh` (existing) |
| Active class | `.btn-auto-refresh--active` (existing) |
| Height | 32px (existing) |
| Horizontal padding | `var(--space-md)` (16px, existing) |
| Border radius | 16px (pill shape, existing) |
| Font size | `var(--font-size-body)` (14px, existing) |
| Font weight | `var(--font-weight-semibold)` (600, existing) |
| Inactive background | `#e0e0e0` (existing) |
| Inactive text | `#666666` (existing) |
| Active background | `var(--color-accent)` (#4a90d9, existing) |
| Active text | `var(--color-text-on-dark)` (#ffffff, existing) |
| Cursor | `pointer` |
| Visibility | Hidden (`display:none`) until connected to a section |

**Label states:**

| State | Button text | Active class? |
|-------|-------------|---------------|
| Auto-refresh off | `Auto` | No |
| Auto-refresh on, first cycle in progress | `Auto` | Yes |
| Auto-refresh on, N cycles completed | `Auto (#N)` | Yes |
| Section switch while auto-on | Reset N to 0, then increment to `#1` after first cycle completes | Yes |

**Interaction:** Clicking toggles `refreshState.active`. When toggled off, `clearTimeout` any pending delay timer. No WebSocket message sent to backend (D-13: backend has no timer to stop).

### 3. Manual Refresh Button (New)

**Source:** D-11

| Property | Value |
|----------|-------|
| Element | `<button id="btn-refresh">` |
| Class | `.btn-refresh` |
| Height | 32px (matches auto-refresh button) |
| Horizontal padding | `var(--space-md)` (16px) |
| Border radius | 16px (pill shape, matches auto-refresh) |

## Interaction States

### Cycle Delay Dropdown

| State | Visual |
|-------|--------|
| Default | White background, grey border, "Continuous" selected |
| Focus | Border color changes to `--color-accent` (#4a90d9) |
| Disabled | Not applicable (dropdown is hidden when not needed, not disabled) |

### Auto-Refresh Button

| State | Visual |
|-------|--------|
| Off | Grey pill (`#e0e0e0` bg, `#666666` text), label: `Auto` |
| On, awaiting first complete | Blue pill (`--color-accent` bg, white text), label: `Auto` |
| On, N cycles done | Blue pill, label: `Auto (#N)` where N increments on each `section_complete` |
| On, section switched | Counter resets to 0; shows `Auto` until first cycle of new section completes, then `Auto (#1)` |

### Manual Refresh Button

| State | Visual |
|-------|--------|
| Idle | Grey pill (`#e0e0e0` bg, `#666666` text), label: `Refresh` |
| Hover | Slightly darker grey (`#d0d0d0` bg) |
| Reading in progress | `opacity: 0.6`, `cursor: not-allowed`, click disabled |
| Hidden | When auto-refresh is on, the Refresh button is not rendered (or `display:none`) |

## Accessibility Notes

- Cycle delay dropdown: native `<select>` element provides keyboard navigation and screen reader support out of the box.
- Auto-refresh button: add `aria-pressed="true"` when active, `aria-pressed="false"` when inactive, to convey toggle state.
- Manual Refresh button: add `aria-busy="true"` while a read cycle is in progress (disabled state), remove when idle.
- All new controls inherit existing font sizes (14px body) which meet WCAG minimum size requirements.
- Color contrast: `#666666` on `#e0e0e0` is 3.9:1 (passes WCAG AA for large text at 14px bold/semibold). White on `#4a90d9` is 4.6:1 (passes WCAG AA).
