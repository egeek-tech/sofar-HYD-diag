**Step 1: Rebuild the tool to pick up Plan 02's new register files**

**Step 2: Run the discovery tool and capture full output**
```bash
./xlsx-discover -xlsx "SOFAR_Modbus_Protocol_English_G3_V1.29(only 3PH).xlsx" > /tmp/discovery-report.txt 2>&1
```

**Step 3: Analyze the report for "missing from probes" entries**
```bash
grep "missing from probes" /tmp/discovery-report.txt
```

This gives the list of registers present in XLSX and/or V1.38 but not yet in the codebase.

**Step 4: For each "missing from probes" register, determine which section it belongs to by address range:**

| Address Range | Target File | Target Variable | Insertion Point |
|---------------|-------------|-----------------|-----------------|
| 0x0400-0x04FF | system.go | SystemGroups | Append to appropriate group or create new group |
| 0x0484-0x04BF | system.go | GridGroups | Append to appropriate phase group |
| 0x0500-0x05FF | system.go (EPSGroups) or pv.go | EPSGroups / GeneratePVGroups | Only if not already covered by dynamic generation |
| 0x0600-0x06FF | battery.go | GenerateBatteryGroups or static group | Only if not covered by dynamic generation |
| 0x5000-0x54FF | dcdc.go | DCDCGroups | Append to appropriate group |
| 0x6000-0x607F | pcu.go | PCUGroups | Append to appropriate group |
| 0x6080-0x60FF | bdu.go | BDUGroups | Append to appropriate group |
| 0x7080-0x70FF | meter.go | MeterGroups | Append (likely already complete) |
| 0x9000-0x91FF | battery.go | BMSInfoGroups | Append if missing |

**Step 5: For each missing register, add it to the appropriate file following the established Probe struct pattern:**
- Use the register name from the discovery tool output (prefer V1.38 name if available, fall back to XLSX name)
- Set data type fields: `IsASCII: true` for ASCII, `U32: true` for U32, `Signed: true` for S16/I16
- Set `Count` appropriately: 1 for U16/S16, 2 for U32, N for ASCII (N registers)
- Set `Unit` and `Scale` from tool output metadata
- Skip registers that are:
  - Already present (the tool shows "all three" or "V1.38+probes")
  - RW or W only (control registers -- per project constraint, read-only tool)
  - Reserved/placeholder registers with no meaningful data
  - In address ranges 0x08xx-0x0Axx (Safety -- deferred to SAFE-01)
  - In address ranges 0x10xx-0x13xx (already covered by Configuration section)
  - In address range 0x20xx (Network/Control -- RW config registers)
  - In address range 0x4Exx (Compatible shared data -- minimal diagnostic value)
  - In address range 0x00xx (General mask areas -- internal protocol, not diagnostic)

**Step 6: For dynamically generated sections (PV, Battery), only add registers that are NOT covered by the generator functions.** The generators cover standard per-channel registers but may miss global/aggregate registers.
