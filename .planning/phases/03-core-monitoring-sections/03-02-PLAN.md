Evolve the hub, message protocol, and API layer to support ProbeGroup-based section registration, grouped data broadcasting, fault reading/decoding, system time composition, PV channel configuration via WebSocket, and the -pv-channels CLI flag. Retire the demo status section.

Purpose: Transform the backend data pipeline from flat key-value pairs to structured grouped data that the frontend (Plan 03) can render as group cards with multi-column layout, fault cards, and PV channel configuration. This is the bridge between the register definitions (Plan 01) and the frontend rendering (Plan 03).

Output: Modified hub package with grouped section support, extended WebSocket message types, configure message handling, CLI flag, and API defaults extension.

    Implement `handleConfigure`:
    ```go
    func (h *Hub) handleConfigure(cmd ClientCommand) {
        msg := cmd.Message
        if msg.Section != "pv" || msg.Config == nil {
            return // silently ignore non-pv configure or missing config
        }

        channels := msg.Config.Channels
        // Clamp to valid range
        if channels < 2 {
            channels = 2
        }
        if channels > 16 {
            channels = 16
        }

        // Rebuild PV section with new channel count
        newGroups := register.GeneratePVGroups(channels)
        sec, ok := h.sections["pv"]
        if !ok {
            return
        }

        // Update the section's groups and flatten probes
        sec.Groups = newGroups
        sec.Probes = flattenProbeGroups(newGroups) // helper function

        h.logger.Info("pv section reconfigured", "channels", channels)

        // Trigger immediate re-read if connected and section has subscribers
        if h.connected && sec.SubscriberCount() > 0 {
            h.triggerSectionRead("pv")
        }
    }
    ```

    Add `flattenProbeGroups` helper (can go in section.go):
    ```go
    func flattenProbeGroups(groups []register.ProbeGroup) []register.Probe {
        var probes []register.Probe
        for _, g := range groups {
            probes = append(probes, g.Probes...)
        }
        return probes
    }
    ```
