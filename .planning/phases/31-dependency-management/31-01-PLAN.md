Create `.github/dependabot.yml` with the following exact content:

```yaml
version: 2

updates:
  # Go module dependencies (per D-01, D-02)
  - package-ecosystem: "gomod"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
      time: "06:00"
      timezone: "UTC"
    groups:
      all-go-deps:
        patterns:
          - "*"
    commit-message:
      prefix: "chore"
      include: "scope"
    labels:
      - "dependencies"
      - "go"

  # GitHub Actions dependencies (per D-03, D-04)
  - package-ecosystem: "github-actions"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
      time: "06:00"
      timezone: "UTC"
    groups:
      all-actions:
        patterns:
          - "*"
    commit-message:
      prefix: "chore"
      include: "scope"
    labels:
      - "dependencies"
      - "github-actions"
```

Key design decisions reflected:
- `groups` with `patterns: ["*"]` batches all updates per ecosystem into one PR (per D-01, D-03)
- No `ignore` or `update-types` filters means all version types including major bumps are included (per D-02)
- `commit-message.prefix: "chore"` with `include: "scope"` produces messages like `chore(deps): bump ...` which release-please ignores (prevents accidental releases from dependency bumps)
- `schedule.interval: "weekly"` on Monday 06:00 UTC (per D-05, day/time is Claude's discretion)
- `directory: "/"` points to repository root where go.mod lives
- Labels added per ecosystem for easy PR filtering ("dependencies" + "go" or "github-actions")
- No `reviewers` or `assignees` set (solo developer, not needed)
- Default `open-pull-requests-limit: 5` is sufficient (grouped updates produce at most 1 PR per ecosystem per week)
