Create `cmd/fyne-poc/main.go` -- a standalone Fyne v2 native desktop application. This is a NEW file in a NEW directory. Do NOT modify any existing files.

**CLI flags** (match the web server pattern from `cmd/server/main.go`):
- `-inverter-host` (default "10.5.99.29") -- inverter IP
- `-inverter-port` (default 4192) -- inverter TCP port
- `-slave` (default 1) -- Modbus slave ID (1-247)
- `-modbus-mode` (default "tcp") -- "tcp" or "rtu"
- `-log-level` (default "info") -- debug/info/warn/error

**Application structure:**

1. **Window setup**: Create a Fyne app and window titled "Sofar HYD - System Monitor (PoC)". Set a reasonable minimum size (700x600).

2. **Connection bar** (top of window):
   - A status label showing current broker state (use broker.StateEvents() channel to update). Color-code: green for "connected", yellow for "connecting"/"reconnecting", red for "disconnected", gray for "dormant".
   - A "Connect" button that calls `broker.Reconfigure(ctx, addr, slaveID)` to initiate connection. When connected, the button text changes to "Disconnect" and calls `broker.Disconnect(ctx)`. Toggle behavior.

3. **System parameters display** (scrollable content area):
   - Iterate over `register.SystemGroups` (Identity, Firmware, Status, Temperatures, Protection).
   - For each ProbeGroup, create a section header (bold label with the group name).
   - For each Probe in the group, create a row with: probe Name (left-aligned label) and a value label (right-aligned, initially showing "---").
   - Store all value labels in a map keyed by probe address (`map[uint16]*widget.Label`) for targeted updates.
   - Special handling for System Time: the 6 time registers (0x042C-0x0431) should have a SINGLE display row labeled "System time" that composes the datetime using `register.ComposeSystemTime()`. Store a reference to this label separately. Still read each register individually but only update the composed time label after all 6 are read.

4. **Per-parameter streaming goroutine**:
   - When connected (watch broker.StateEvents()), start a polling loop goroutine.
   - The loop iterates over ALL probes in `register.SystemGroups` sequentially.
   - For each probe, call `broker.ReadRegisters(ctx, probe.Addr, probe.Count)`.
   - On success, format with `register.FormatValue(probe, data)` and update THAT probe's label immediately via the Fyne label's `SetText()` method. Because Fyne's widget.Label.SetText() is thread-safe, this works from goroutines without explicit locking.
   - On error, set the label text to the error string (e.g., "read error: timeout").
   - For System Time probes: accumulate the 6 values, then after the last one (Sec at 0x0431), compose and update the single time label.
   - After completing all probes, loop back and start again (continuous polling).
   - When disconnected, stop the loop (use context cancellation). Reset all value labels to "---".
   - The broker's internal 500ms inter-read delay handles Modbus timing -- no extra sleeps needed between reads.

5. **Graceful shutdown**: Handle SIGINT/SIGTERM to cancel the context, close the broker, and exit cleanly.

**Layout**: Use `container.NewBorder()` with the connection bar at top, and a `container.NewVScroll()` wrapping a `container.NewVBox()` of all the group sections in the center. Each group section: a bold header label followed by a `widget.Form` or `container.NewGridWithColumns(2)` for name-value pairs.

**Important implementation details:**
- Use `fyne.io/fyne/v2/app`, `fyne.io/fyne/v2/widget`, `fyne.io/fyne/v2/container`, `fyne.io/fyne/v2/layout`, `fyne.io/fyne/v2/theme` packages.
- The broker's `ReadRegisters` already enforces 500ms inter-read delays, so each probe read takes ~500ms minimum. With ~20 probes in SystemGroups, a full cycle takes ~10s. This is the desired streaming effect -- the user sees values appearing one-by-one.
- Use `encoding/binary` for extracting uint16 values from the time register bytes when accumulating for ComposeSystemTime.
- Start the broker with `go b.Run(ctx)` in main, same as the web server does.
- The polling goroutine should check `ctx.Done()` between each read to allow clean shutdown.
- When the connect/disconnect button is clicked, use a separate goroutine for the broker call to avoid blocking the UI thread.
